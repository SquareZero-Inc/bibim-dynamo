# BIBIM AI v2.3.10 릴리즈 노트

**릴리즈일**: 2026-03-20

## 🔧 구조적 안정성 강화 + 기술 부채 정리

### Spec 기반 코드 생성 경로 안정화
- 코드 생성 요청 판정 방식을 **언어 무관 구조화 토큰**으로 전환했습니다. 이전에는 한국어/영어 마커 문자열에 의존하여 인코딩이나 언어 변경 시 감지가 실패할 수 있었습니다.
- 이 변경으로 **영어 모드에서 RAG/분석 컨텍스트가 누락되는 문제**가 구조적으로 재발할 수 없게 됩니다.

### RAG 실패 시 사용자 알림
- API 문서 검색(RAG)이 실패하면 **사용자에게 경고 메시지**가 표시됩니다. 이전에는 실패가 조용히 무시되어 코드 품질 저하 원인을 알 수 없었습니다.
- 일시적 오류(타임아웃, 서버 에러)에 한해 **자동 재시도** 후 실패하면 경고를 표시합니다.
- "검색 결과 없음"(no_match)은 정상 동작이므로 재시도하지 않습니다.

### 코드베이스 정리
- 미사용 Legacy 코드(5개 파일)와 불필요한 Wrapper 클래스(3개 파일)를 삭제하여 코드베이스를 경량화했습니다.
- `GeminiService.cs`의 인코딩 깨진 주석 19줄을 정리했습니다.
- 프로젝트 구조가 더 명확해져 유지보수성이 향상됩니다.

### API 검증 리플레이 회귀 테스트
- 실제 유저 실패 사례 기반 **19개 리플레이 테스트**를 추가했습니다.
- Block(제거된 API, 존재하지 않는 enum/멤버), Pass(정상 코드), Warn(deprecated), Edge case(중첩 괄호, auto-fix) 시나리오를 커버합니다.
- 검증 로직 수정 시 기존 동작이 깨지지 않는지 자동으로 확인할 수 있습니다.

### GeminiService 프롬프트 분리
- 코드 생성 시스템 프롬프트(~130줄), RAG 쿼리 프롬프트(~40줄), RAG 검증 프롬프트(~80줄)를 `Services/Prompts/` 폴더로 분리했습니다.
- 프롬프트 수정 시 git diff 가독성이 향상되고, 프롬프트만 독립적으로 롤백할 수 있습니다.

### 품질 관리 인프라
- **실패 기록부**: 과거 19개 버그 사례와 금지 패턴을 문서화하여 같은 실수 반복을 방지합니다.
- **배포 전 체크리스트**: 4단계 검증 프로세스(빌드/API 확인/수동 테스트/배포)를 표준화했습니다.
- **자동 빌드 체크**: 코드 수정 시 자동으로 컴파일 에러를 감지합니다.
